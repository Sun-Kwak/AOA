# Prompt

## Purpose

AIOS의 에이전트 정의 파일에 프롬프트를 작성할 때의 규칙을 정의한다.
일관된 프롬프트 구조가 있어야 에이전트 품질이 예측 가능하고
재사용성이 높아진다.

---

## Scope

공용 에이전트(`Agents/`), 프로젝트 에이전트(`Projects/<name>/Agents/`),
서브에이전트의 정의 파일에 포함되는 모든 프롬프트에 적용된다.

---

## 에이전트 프롬프트 구조

모든 에이전트 프롬프트는 다음 순서를 따른다:

```
1. Role       — 이 에이전트가 누구인지
2. Context    — 어떤 상황에서 동작하는지
3. Task       — 무엇을 해야 하는지
4. Rules      — 반드시 따라야 하는 제약
5. Output     — 출력 형식과 기준
6. Prohibited — 절대 하면 안 되는 것
```

---

## 섹션별 작성 규칙

### 1. Role (역할 선언)
- 첫 문장에서 역할을 명확히 선언한다.
- "당신은 X입니다" 또는 "You are X" 형식.
- 모호한 역할 금지: "당신은 도움이 되는 AI입니다" ❌

```
✅ 당신은 AIOS 프레임워크에서 동작하는 YouTube 쇼츠 스크립트 작성 에이전트입니다.
❌ 당신은 콘텐츠 제작을 도와주는 에이전트입니다.
```

---

### 2. Context (컨텍스트)
- 어떤 프로젝트에서, 어떤 목적으로 호출되는지 명시.
- AIOS 프레임워크 규칙을 따른다는 것을 명시.
- 과도한 배경 설명 금지 — 핵심만.

---

### 3. Task (태스크 정의)
- 이 에이전트가 받는 입력과 생성해야 하는 출력을 명확히.
- 동사로 시작하는 지시문 형식.
- 여러 태스크가 있으면 번호로 나열.

```
✅ 1. 입력된 주제로 60초 분량의 스크립트를 작성한다.
   2. 스크립트는 후크 → 본문 → CTA 구조를 따른다.
```

---

### 4. Rules (제약 규칙)
- 에이전트가 반드시 따라야 하는 규칙만 포함.
- AIOS Core/Policies는 여기서 반복하지 않는다.
  (이미 상속됨)
- 이 에이전트에만 적용되는 규칙만 작성.

---

### 5. Output (출력 형식)
- 출력 형식을 반드시 명시한다.
- 포맷: Markdown / YAML / JSON / Plain Text 중 하나.
- 예시 출력을 포함하면 더 좋다.

```
출력 형식:
  ```markdown
  ## 제목
  **후크**: ...
  **본문**: ...
  **CTA**: ...
  ```
```

---

### 6. Prohibited (금지 사항)
- 이 에이전트가 절대 해서는 안 되는 것.
- AIOS Safety Policy와 중복이어도 에이전트 레벨에서 명시.
- 명확한 금지 동사로 작성.

```
✅ 절대 Framework 파일을 수정하지 않는다.
✅ 사용자 확인 없이 외부 API를 호출하지 않는다.
❌ 부적절한 내용은 생성하지 마세요. (너무 모호함)
```

---

## 프롬프트 길이 가이드

| 에이전트 유형 | 권장 길이 |
|--------------|-----------|
| 단순 툴 래퍼 에이전트 | 200자 이내 |
| 일반 공용 에이전트 | 300~600자 |
| 복잡한 프로젝트 에이전트 | 600~1000자 |
| 최대 | 1500자 (초과 시 분리 검토) |

프롬프트가 길수록 토큰 비용 증가.
불필요한 배경 설명, 반복, AIOS 공통 규칙 재서술은 제거.

---

## 언어 규칙

- 에이전트 프롬프트 언어는 프로젝트 기본 언어를 따른다.
- 공용 에이전트는 영문 우선.
- 한국어 전용 프로젝트의 프로젝트 에이전트는 한국어 허용.
- Role 선언은 어떤 언어든 명확해야 한다.
