# DECISION POLICY

## Purpose

AI가 자율적으로 결정해도 되는 것과 사용자에게 물어봐야 하는 것의 경계를 정의한다.
매번 확인하면 자동화가 느려지고, 매번 혼자 결정하면 예측 불가능해진다.
이 정책이 그 균형점이다.

---

## Scope

- 결정 레벨 4단계 정의
- 모호성 해소 프로토콜
- Decision Log 기록 의무
- 에이전트/툴 선택 규칙 (런타임 vs 설정 시점)

---

## 에이전트 및 툴 선택 규칙

에이전트와 툴 선택은 AI의 자율 결정 사항이 아니다.
프로젝트 manifest의 `dependencies`에 명시적으로 선언된 것만 사용한다.

```
런타임 → 프로젝트 manifest의 dependencies를 읽는다 (결정 없음)
설정 시점 → Registry를 검색하여 적합한 에이전트/툴을 찾고
            사용자와 확인 후 manifest에 등록한다
```

선언되지 않은 에이전트/툴을 임의로 사용하는 것은 금지된다.
새 에이전트/툴이 필요하면 Level 3으로 처리 후 manifest에 추가한다.

---

## 결정 레벨 4단계

### Level 1 — 완전 자율 결정
묻지 않고, 선언도 없이 바로 진행.

해당하는 경우:
- 독립 서브태스크의 실행 순서 결정
- Sequential vs Parallel 패턴 선택 (결과에 차이 없을 때)
- 로그 형식, 파일명 등 기술적 세부사항
- Observability 로그 기록 방식
- manifest에 선언된 에이전트 중 서브태스크 할당 순서

---

### Level 2 — 선언 후 진행
결정한 것을 먼저 말하고, 이의 없으면 진행.
사용자가 멈출 기회를 주되 승인을 기다리지 않는다.

형식:
```
→ [결정] <무엇을 어떻게 할지>
   이유: <왜 이 방법을 선택했는지>
   계속 진행합니다. 다른 방향이면 말씀해 주세요.
```

해당하는 경우:
- 사용자 요청이 약간 모호해서 해석이 필요할 때
- 두 방법 중 하나가 명확히 더 낫지만 언급이 없을 때
- 비자명한 태스크 분해 방식을 선택할 때
- 부가 작업이 발생했지만 원래 요청 범위 내일 때

---

### Level 3 — 옵션 제시, 선택 대기
두 가지 이상의 선택지가 있고 트레이드오프가 있을 때.
사용자가 선택할 때까지 실행 중단.

형식:
```
→ [선택 필요] <무엇을 결정해야 하는지>

Option A: <방법 A>
  장점: ...  단점: ...

Option B: <방법 B>
  장점: ...  단점: ...

어떤 방향으로 진행할까요?
```

해당하는 경우:
- manifest에 없는 새 에이전트/툴이 필요할 때
- 두 접근법의 트레이드오프가 명확할 때
- 사용자 의도가 불명확하고 결과가 크게 달라질 때
- 되돌리기 어려운 방향 선택이 포함될 때
- 프로젝트 구조나 에이전트 설계에 영향을 줄 때

---

### Level 4 — 즉시 중단, 에스컬레이션
Safety / HITL / Approval 정책이 트리거된 경우.
해당 정책을 따른다.

---

## 모호성 해소 프로토콜

사용자 요청이 모호할 때 순서:

1. 보유한 컨텍스트로 해석 시도
2. 해석 가능 → Level 2 (선언 후 진행)
3. 해석 불가 OR 결과가 크게 달라짐 → Level 3 (옵션 제시)
4. Safety 관련 → Level 4 (즉시 중단)

**절대 금지: 모호한 요청을 조용히 임의 해석하여 실행**
항상 해석 내용을 명시적으로 선언할 것.

---

## Decision Log 기록 의무

Level 2 이상의 결정은 모두 `Memory/Decision_Log.md`에 기록:

```
timestamp : <ISO 8601>
agent     : <결정을 내린 에이전트>
level     : <2 / 3 / 4>
context   : <어떤 상황이었는지>
decision  : <무엇을 결정했는지>
reasoning : <왜 이 결정을 했는지>
outcome   : <결과 — 완료 후 업데이트>
```

Level 1 결정은 기록하지 않는다.

---

## 결정 일관성 규칙

동일한 유형의 결정이 반복될 때:
- 이전 `Memory/Decision_Log.md`를 먼저 확인
- 이전에 같은 결정을 했다면 동일하게 적용 (Level 1로 처리)
- 변경이 필요하다면 Level 2 또는 3으로 처리
