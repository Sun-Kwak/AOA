## [Workflow-001] 콘텐츠 변형 단계 누락

**발생일**: 2026-07-15  
**Phase**: Phase 1 → Phase 2 (트렌드 수집 → 이미지 생성)  
**상황**: 테스트 실행 중 image_edit 모드 사용  

**실수**:  
트렌드 수집 후 바로 image-generator에게 프롬프트 전달 시, **"맥락 유지 + 내용 변경"** 의도를 제대로 전달하지 못함.

예시:
- 원본 트렌드: "축복받은 유전자 체크리스트"
- 잘못된 요청: "아침 공복 운동 vs 식후 운동" (완전히 다른 주제로 변경)
- 올바른 요청: "축복받은 유전자 테스트" (같은 맥락, 세부 항목만 변경)

**결과**:  
- image-generator 세션 중단
- 사용자가 의도와 다른 결과 예상
- 수동 프롬프트 작성 시 반복적 실수 위험

**회피전략**:  
**Content Transformer Agent 추가 (공용 에이전트)**

새로운 워크플로우:
```
Phase 1: 트렌드 수집
  ↓
Phase 1.5: 콘텐츠 변형 ✨ (NEW)
  - 이미지/텍스트 분석
  - 맥락(주제, 형식) 파악
  - 세부 내용만 변형
  - image_edit 프롬프트 자동 생성
  ↓
Phase 2: 이미지 생성
  - Phase 1.5의 프롬프트 사용
```

**구현 위치**: `Registry/Agents/content-transformer.md` (공용)  
**사용 시점**: 트렌드 수집과 이미지 생성 사이  
**효과**: "맥락 유지" 로직을 자동화하여 수동 실수 방지
