# Project Wiki

## Purpose

health-fitness-cards 프로젝트 실행 중 발견한 실수와 해결책을 기록한다.

프로젝트별 Wiki는 워크플로우 조율, 에이전트 간 데이터 전달, 출력 처리에 특화된다.

---

## Wiki 파일

- **workflow_execution.md**: 워크플로우 실행 순서, Phase 전환 실수
- **agent_coordination.md**: 에이전트 세션 생성, 병렬 실행 조율
- **output_handling.md**: 결과 저장, 파일명 충돌, 메타데이터

---

## 사용 규칙

### 프로젝트 시작 시
1. 모든 Wiki 문서 읽기
2. 기록된 실수 패턴 확인
3. 회피 전략 적용

### 프로젝트 완료 후
1. 새로운 실수 발견 시 즉시 기록
2. 패턴 번호 부여
3. 회피 전략 명시

---

## 누적 효과

- 이 프로젝트가 반복 실행될 때마다 개선
- 워크플로우 안정성 향상
- 에이전트 조율 효율 증가
