# LLM Wiki

## Purpose

AI가 작업 중 발견한 실수, 해결책, 패턴을 자동으로 기록하고 재사용하는 지식베이스.

Wikipedia처럼 AI가 작성자이자 독자이며, 세션 간 학습이 누적된다.

---

## Scope

이 디렉터리는 AI의 **누적 학습 기록**을 저장한다:

- **실수 패턴**: 반복된 실수와 발생 원인
- **회피 전략**: 실수를 방지하는 구체적 절차
- **작동 패턴**: 성공한 워크플로우 패턴
- **의사결정 기준**: 애매한 상황에서의 판단 기준

---

## 구조

```
Memory/Wiki/
  ├── README.md                    ← 이 문서
  ├── project_creation.md          ← 프로젝트 생성 시 실수/패턴
  ├── agent_creation.md            ← 에이전트 생성 시 실수/패턴  
  ├── session_management.md        ← 세션 생성 시 실수/패턴
  └── registry_management.md       ← Registry 업데이트 실수/패턴
```

---

## 사용 규칙

### 작업 시작 시 (필수)

해당 작업의 Wiki 문서를 **반드시 먼저 읽는다**:

- 프로젝트 생성 요청 → `project_creation.md` 조회
- 에이전트 생성 요청 → `agent_creation.md` 조회
- 세션 생성 요청 → `session_management.md` 조회
- Registry 수정 → `registry_management.md` 조회

Wiki에 기록된 실수 패턴을 확인하고 회피 전략을 적용한다.

### 작업 완료 시

새로운 실수나 해결책을 발견하면 **즉시 해당 Wiki 문서에 추가**한다:

```markdown
## [Pattern-XXX] 패턴 이름

**발생일**: YYYY-MM-DD
**상황**: 어떤 상황에서 발생했는가
**실수**: 무엇을 잘못했는가
**결과**: 어떤 문제가 생겼는가
**회피전략**: 다음부턴 어떻게 해야 하는가

---
```

---

## 원칙

1. **누적식 지식베이스**: 삭제 금지, 추가만 허용
2. **구체적 기록**: 추상적 조언이 아닌 구체적 절차
3. **즉시 업데이트**: 실수 발견 즉시 기록 (나중에 X)
4. **필수 조회**: 작업 전 Wiki 확인은 선택이 아닌 의무
